# IMSAM 면접 평가 루브릭

## 개요
이 문서는 AI 면접관이 지원자의 답변을 평가할 때 사용하는 기준입니다.
각 항목은 1-5점 척도로 평가하며, 점수별 기준을 참고하여 일관된 평가를 수행합니다.

---

## 1. 논리적 구조 (가중치: 20%)

**설명:** STAR(상황-과제-행동-결과) 또는 PREP(결론-이유-예시-결론) 구조로 체계적으로 답변했는가

### 점수 기준
| 점수 | 기준 |
|------|------|
| 1점 | 답변이 산만하고 구조가 없음. 질문과 관련 없는 내용이 많음 |
| 2점 | 답변에 일부 구조가 있으나 논리적 흐름이 부족함 |
| 3점 | 대체로 구조적이나 세부 연결이 약함. STAR 일부 요소 누락 |
| 4점 | 명확한 구조로 답변. STAR/PREP 대부분 충족 |
| 5점 | 매우 체계적이고 논리적인 구조. 완벽한 STAR/PREP 적용 |

### 좋은 답변 예시
> 해당 프로젝트에서 (상황) 서버 응답 속도가 3초 이상 걸리는 문제가 있었습니다. (과제) 저는 성능 최적화를 담당했고, (행동) 쿼리 분석 후 인덱스 추가와 캐싱 레이어를 도입했습니다. (결과) 응답 속도가 0.3초로 90% 개선되었습니다.

### 나쁜 답변 예시
> 음... 그 프로젝트에서 뭔가 느렸던 것 같아요. 그래서 이것저것 해봤는데... 잘 됐던 것 같습니다.

### 긍정적 키워드
우선, 첫째, 둘째, 결론적으로, 요약하면, 상황은, 과제는, 제가 한 행동은, 결과는, 예를 들어, 구체적으로, 그 결과로

### 부정적 키워드
음..., 그냥, 뭐랄까, 아무튼, 글쎄요, 잘 모르겠지만, 갑자기 생각나는 건

## 2. 직무 전문성 (가중치: 30%)

**설명:** 직무 관련 지식, 경험 사례, 문제 해결 방식이 구체적인가

### 점수 기준
| 점수 | 기준 |
|------|------|
| 1점 | 직무 관련 지식이 거의 없음. 경험 사례가 모호하거나 없음 |
| 2점 | 기본 개념은 알지만 깊이가 부족함. 경험이 피상적 |
| 3점 | 직무 지식과 경험이 있으나 구체성이 부족함 |
| 4점 | 충분한 직무 역량 보유. 도구, 방법론을 구체적으로 설명 |
| 5점 | 깊은 전문성. 지표, 수치, 트레이드오프까지 상세히 설명 |

### 좋은 답변 예시
> TypeScript와 React를 사용해 컴포넌트 설계 시 합성 패턴을 적용했습니다. 이를 통해 재사용성을 높이고, 번들 사이즈를 30% 줄였습니다. 특히 lazy loading과 code splitting을 활용해 초기 로딩 시간을 2초에서 0.8초로 개선했습니다.

### 나쁜 답변 예시
> 프론트엔드 개발 했습니다. React 썼고요. 잘 됐습니다.

### 긍정적 키워드 (개발 직군)
API, REST, GraphQL, 마이크로서비스, 스케일링, CI/CD, Docker, Kubernetes, AWS, GCP, TDD, 코드 리뷰, 리팩토링, 아키텍처

### 성과 지표 키워드
00% 개선, 00% 감소, DAU, MAU, CTR, CVR, latency, throughput, SLA, uptime

### 부정적 키워드
잘 모르겠습니다, 안 해봤습니다, 들어는 봤는데, 그건 다른 팀에서, 제 담당이 아니었어요

## 3. 태도/커뮤니케이션 (가중치: 20%)

**설명:** 말하기 태도, 명료성, 예의, 협업 자세가 적절한가

### 점수 기준
| 점수 | 기준 |
|------|------|
| 1점 | 부정적 인상. 무례하거나 방어적. 소통이 원활하지 않음 |
| 2점 | 소극적이거나 자신감 부족. 소통에 어려움 |
| 3점 | 무난한 수준. 기본적인 예의와 소통 능력 보유 |
| 4점 | 긍정적이고 적극적. 명확하게 의사 표현 |
| 5점 | 매우 긍정적 인상. 뛰어난 소통 능력과 협업 자세 |

### 좋은 답변 예시
> 좋은 질문 감사합니다. 해당 프로젝트에서 팀원들과 적극적으로 소통하며 문제를 해결했습니다. 특히 디자이너분과의 협업에서 일일 싱크업을 통해 피드백을 빠르게 반영했고, 이 경험을 통해 커뮤니케이션의 중요성을 배웠습니다.

### 나쁜 답변 예시
> 그건 다른 팀원이 잘못해서 그런 거예요. 저는 제 일만 했는데 왜 물어보시는지...

### 긍정적 키워드
감사합니다, 좋은 질문입니다, 말씀해 주신 것처럼, 함께, 협업, 팀원들과, 공유했습니다, 피드백을 반영해, 배웠습니다, 성장했습니다, 개선했습니다

### 부정적 키워드
그건 제 잘못이 아니에요, 다른 사람이, 저만 제대로, 몰라서, 시키는 대로, 어쩔 수 없이

## 4. 회사/직무 적합도 (가중치: 15%)

**설명:** 조직의 가치, 직무 요구사항과의 연결 정도

### 점수 기준
| 점수 | 기준 |
|------|------|
| 1점 | 회사/직무에 대한 이해 없음. 연결점 전혀 없음 |
| 2점 | 피상적인 이해. 일반적인 답변만 제시 |
| 3점 | 기본적인 이해. 일반적인 수준의 연결 |
| 4점 | 회사/직무를 잘 이해하고 자신의 경험과 연결 |
| 5점 | 깊은 이해. 구체적인 기여 방안과 비전 제시 |

### 좋은 답변 예시
> 귀사가 AI 기반 서비스 확장에 집중하고 있다고 알고 있습니다. 제가 이전 회사에서 ML 파이프라인 구축 경험이 있어서, 이 포지션에서 데이터 처리 자동화에 기여하고 싶습니다. 특히 귀사의 "데이터 민주화" 비전에 공감합니다.

### 나쁜 답변 예시
> 음... 이 회사가 뭐 하는 곳인지 정확히는 모르겠는데, 일단 개발자 뽑는다고 해서요.

### 긍정적 키워드
귀사의, 이 회사의, 이 포지션에서, 조직의 미션, 회사의 비전, 팀의 목표, 기여할 수 있는, 성장할 수 있는

### 부정적 키워드
아무 회사나, 어디든, 월급만 주면, 잘 모르겠는데, 조사는 안 해봤지만

## 5. 성장 가능성 (가중치: 15%)

**설명:** 피드백 수용 태도, 자기 성찰, 학습 의지가 드러나는가

### 점수 기준
| 점수 | 기준 |
|------|------|
| 1점 | 성장 의지 없음. 피드백 거부. 자기 성찰 없음 |
| 2점 | 소극적. 피드백 수용 의지 약함 |
| 3점 | 일부 성장 의지 언급. 기본적인 자기 성찰 |
| 4점 | 적극적인 학습 의지. 실패에서 배운 점 명확 |
| 5점 | 뛰어난 성장 마인드셋. 구체적인 학습 계획과 자기 객관화 |

### 좋은 답변 예시
> 처음 TDD를 도입했을 때 팀에서 저항이 있었습니다. 제가 너무 급하게 추진한 것이 원인이었다고 생각합니다. 이 경험에서 변화 관리의 중요성을 배웠고, 이후에는 작은 범위부터 점진적으로 도입하는 방식으로 개선했습니다.

### 나쁜 답변 예시
> 저는 실패한 적이 없어요. 항상 다 잘했습니다.

### 긍정적 키워드
배웠습니다, 성장했습니다, 개선 중입니다, 피드백을 받고, 부족한 점을 깨닫고, 도전하고 있습니다, 새롭게 배우고 싶은, 더 나아지기 위해, 실패했지만, 그 경험을 통해, 앞으로는

### 부정적 키워드
더 배울 건 없는 것 같고, 이미 다 알고, 실패한 적 없습니다, 완벽하게, 제가 최고

| 구분 | 점수 범위 | 설명 |
|------|----------|------|
| **합격** | 70점 이상 | 채용 권고 |
| **보류** | 50-69점 | 추가 검토 필요 |
| **불합격** | 50점 미만 | 채용 비권고 |

## 최종 점수 산정 방법

```
총점 = Σ(항목별 점수 × 가중치) × 20
```

- 논리적 구조: 20%
- 직무 전문성: 30%
- 태도/커뮤니케이션: 20%
- 회사/직무 적합도: 15%
- 성장 가능성: 15%

**예시:**
- 논리적 구조: 4점 × 0.20 = 0.8
- 직무 전문성: 4점 × 0.30 = 1.2
- 태도/커뮤니케이션: 5점 × 0.20 = 1.0
- 회사/직무 적합도: 3점 × 0.15 = 0.45
- 성장 가능성: 4점 × 0.15 = 0.6

가중 합계: 4.05점 (5점 만점)
100점 환산: (4.05 / 5) × 100 = **81점 → 합격**

## 면접관별 평가 중점

### 실무팀장 (Hiring Manager)
- **직무 전문성** 45% (가장 중시)
- 성장 가능성 15%
- 논리적 구조 15%
- 태도/커뮤니케이션 15%
- 회사/직무 적합도 10%

### HR 담당자 (HR Manager)
- **태도/커뮤니케이션** 35% (가장 중시)
- **회사/직무 적합도** 25% (중시)
- 직무 전문성 15%
- 성장 가능성 15%
- 논리적 구조 10%

### 시니어 동료 (Senior Peer)
- **직무 전문성** 35% (가장 중시)
- **성장 가능성** 25% (중시)
- 논리적 구조 20%
- 태도/커뮤니케이션 15%
- 회사/직무 적합도 5%

## 8축 역량과의 매핑

5축 핵심 평가 항목은 8축 역량 점수로 변환됩니다:

| 핵심 평가 | → | 8축 역량 |
|----------|---|---------|
| 논리적 구조 | → | 명확성(40%), 논리적 사고(40%), 커뮤니케이션(20%) |
| 직무 전문성 | → | 문제 해결(40%), 이해력(30%), 논리적 사고(20%), 적응력(10%) |
| 태도/커뮤니케이션 | → | 커뮤니케이션(40%), 행동 역량(30%), 리더십(20%), 적응력(10%) |
| 회사/직무 적합도 | → | 적응력(40%), 행동 역량(30%), 이해력(30%) |
| 성장 가능성 | → | 적응력(40%), 행동 역량(30%), 리더십(20%), 문제 해결(10%) |

---

## 사용 방법

1. **실시간 평가**: 각 답변마다 5개 항목을 1-5점으로 평가
2. **점수 근거 기록**: 점수의 이유를 답변 내용에서 구체적으로 인용
3. **피드백 제공**: 강점과 개선점을 함께 제시
4. **최종 종합**: 면접 종료 후 전체 답변을 종합하여 최종 평가

이 루브릭을 기준으로 면접 답변을 평가하면 일관성 있고 객관적인 채점이 가능합니다.
